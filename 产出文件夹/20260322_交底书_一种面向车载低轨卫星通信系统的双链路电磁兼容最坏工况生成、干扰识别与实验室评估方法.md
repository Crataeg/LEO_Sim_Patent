## 二、介绍现有技术并说明现有技术存在的问题（痛点）
低轨卫星通信正在成为车载超视距通信的重要补充，但现有车载卫星通信专利和产品方案大多聚焦于终端结构、卫星接入流程、数据压缩、天线姿态控制、收发能力检测或者现场故障定位，缺少一套面向研发前期和实验室阶段的系统级电磁兼容评估方法。尤其当系统进入车载应用场景后，通信链路不再是静态、单链路、单干扰源问题，而是同时受到轨道几何时变、星地可见性变化、频率复用带来的共信道干扰、多干扰源耦合、抗干扰动作延迟以及上行与下行链路非对称退化的共同影响。

现有实验室 EMC 测试方式通常存在以下不足。

1. 以固定工况或少量人工枚举工况为主，难以覆盖真实低轨星座场景下的时变几何关系，无法准确反映车载用户终端、网关地面站、服务卫星以及干扰卫星之间的动态耦合关系。
2. 大多只评估单一链路，例如只看下行接收性能或者只看某一端口的静态抗扰度，没有把上行链路、下行链路和端到端业务性能放到同一评价闭环中，容易遗漏实际瓶颈。
3. 现有方法通常只能得到“链路变差了”这一结果，缺少对干扰类型的判别，无法区分是单音干扰、部分带噪声压制干扰、调制型干扰还是共信道干扰主导，因此不利于后续抗干扰措施选型。
4. 采用全波形、全链路精细仿真时，计算量大、场景构造重，难以开展大批量最坏工况搜索；而采用纯规则方法时，又容易漏掉真正导致系统退化的隐蔽工况。
5. 现有车载天线故障现场诊断类方案主要解决现场测量与故障定位问题，无法替代研发阶段对“系统在什么干扰轨迹下会失效、何时失效、由何类干扰触发失效”的实验室前置评估。
6. 现有电磁环境建模类方案多面向近场等效建模或静态场预测，缺乏与低轨星座时变几何、链路复用、抗干扰动作和业务级 KPI 相结合的闭环设计。

因此，迫切需要一种新的技术方案，将车载低轨卫星通信系统的双链路通信、低轨星座几何变化、共信道干扰、外部干扰、最坏工况自动搜索以及干扰类型识别耦合到同一实验室评估方法中，在计算复杂度可控的前提下输出对研发设计真正有指导意义的指标和图像。

## 三、介绍本发明解决的问题及产生的有益效果
本发明要解决的核心技术问题是：如何在不依赖大量实轨试验和全波形重仿真的前提下，构建一套能够面向车载低轨卫星通信系统进行双链路电磁兼容最坏工况生成、干扰识别和实验室评估的技术方案，使研发人员能够在实验室阶段就发现真正影响系统可靠性的关键工况，并得到可追溯的指标和图像证据。

本发明通过“时变场景构建 + 双链路统一建模 + 生成式最坏工况搜索 + 时频图干扰识别 + 合规性输出”的组合方案，获得如下有益效果。

1. 将用户终端、网关、服务卫星和干扰卫星放入同一时变场景中，能够更真实地复现实验室评估对象在动态轨道环境中的电磁兼容风险。
2. 以双链路方式同步计算上行和下行性能，并以 `min{THR_DL, THR_UL}` 形成端到端吞吐量评价值，可直接识别哪一条链路成为系统瓶颈，避免单链路判断失真。
3. 通过生成模型与搜索算法在潜空间和干扰尺度空间中自动寻找最坏工况，摆脱人工枚举场景的低覆盖率问题，使平台能够主动“找出最坏情况”，而不是被动等待测试人员构造场景。
4. 通过功率对齐的 IQ 采样、短时傅里叶变换图像化和轻量卷积神经网络分类，除给出性能下降结果外，还能给出对应的干扰形态标签和关键帧图像，提升结果可解释性。
5. 采用系统级功率仿真与指标代理模型，相比直接进行大规模全波形链路仿真，计算量显著降低，更适合进行工程化批量评估和最坏工况搜索。
6. 保留接收机端口功率、天线参数、天线因子、线损和前放增益等后续对接接口，便于后续从纯仿真阶段平滑过渡到半实物或实验室实测阶段。

## 四、详细说明所采取的技术方案及如何产生有益效果

### 4.1 总体技术思路
本发明提供一种面向车载低轨卫星通信系统的双链路电磁兼容最坏工况生成、干扰识别与实验室评估方法。该方法以车载用户终端、网关地面站、低轨卫星星座和一个或多个干扰卫星为对象，先构建时变几何场景，再对下行链路和上行链路进行统一建模与求解，随后利用生成式干扰包络和搜索算法自动寻找最坏工况，在最坏工况下输出链路指标、端到端指标、干扰类型标签以及关键帧时频图，最终形成适用于实验室测试评估和专利支撑的图文结果。

本发明同时保护方法、系统、电子设备和计算机可读存储介质四种主题。

### 4.2 系统组成
本发明对应的系统至少包括如下模块。

1. 场景构建模块，用于构建卫星场景、车载用户终端位置、网关地面站位置、低轨星座参数和干扰卫星参数。
2. 几何预计算模块，用于计算不同时间步下的方位角、俯仰角、距离、可见性和径向速度。
3. 频率复用与星间路由模块，用于生成频率复用计划和星间链路图，为共信道干扰评估与路由时延估计提供支撑。
4. 双链路功率建模模块，用于分别计算下行链路和上行链路的信号功率、共信道干扰功率、噪声功率和干扰功率。
5. 最坏工况生成模块，用于基于生成模型产生时变干扰包络，并通过搜索算法寻找使系统退化最明显的最坏工况。
6. 双链路协同评估模块，用于在同一路由或同一服务卫星约束下联合求解上行、下行与端到端指标。
7. 干扰识别模块，用于将链路功率级转换为功率对齐的 IQ 快照，再将 IQ 快照转换为时频图，并输出干扰类型分类结果。
8. 结果输出模块，用于输出摘要文本、关键指标、关键帧图像、训练样例拼图、测试集混淆矩阵和附图素材。

### 4.3 方法步骤
本发明的方法优选包括如下步骤。

#### 步骤 S1：构建时变低轨卫星通信评估场景
根据预设纪元时刻、采样时间、仿真时长、星座轨道高度、轨道倾角、轨道面数量、每轨卫星数量、相位参数和最小仰角，构建低轨星座场景；同时设置车载用户终端对应的地面站位置、网关地面站位置以及一个或多个干扰卫星。

优选地，场景中至少包括：

1. 用户地面站，代表车载用户终端。
2. 网关地面站，代表回传侧地面接入节点。
3. 低轨卫星星座，代表服务网络。
4. 干扰卫星，代表外部干扰源或压制源。

通过该步骤可以把传统静态 EMC 工况转换为时变几何工况，为后续所有链路退化、共信道干扰和抗干扰事件提供统一时轴。

#### 步骤 S2：预计算几何关系和频率复用关系
对每一时间步，分别计算用户终端与各卫星之间、网关地面站与各卫星之间以及用户终端与各干扰卫星之间的方位角、俯仰角、距离和可见性；同时计算服务卫星的径向速度以用于多普勒估计。

在此基础上，按照轨道面编号和卫星槽位编号生成频率复用通道编号，并构建简化的星间链路图，用于后续求解服务卫星到网关卫星之间的跳数和传播时延。

该步骤的有益效果在于：

1. 共信道干扰不再是静态常量，而是随时间步和频率复用关系动态变化。
2. 上行和下行链路可在同一几何和复用框架下求解，保证评价口径一致。

#### 步骤 S3：构建双链路功率模型
分别针对下行链路和上行链路计算：

1. 服务信号功率。
2. 共信道干扰功率。
3. 噪声功率。
4. 干扰卫星到用户终端或上行代理端口的等效干扰功率。

优选地，干扰功率通过干扰卫星与服务卫星之间的离轴角计算方向图衰减，并结合主瓣、旁瓣和底噪增益进行修正。上行链路可以采用固定干扰代理、复用下行链路干扰代理或无干扰代理三种方式之一。

该步骤的有益效果在于：把传统“只看接收端一个功率值”的方式扩展成可解释的功率分解结构，为后续识别干扰主导因素和生成时频图提供基础。

#### 步骤 S4：建立双链路统一指标代理模型
基于步骤 S3 的功率模型，在每一时间步上求解：

1. 信干噪比。
2. 误码率。
3. 吞吐量。
4. 误块率。
5. 多普勒及多普勒变化率。
6. 星间路由跳数和端到端传播时延。
7. 事件标签，包括正常、压制干扰、受保护、共信道和无服务等状态。

优选地，端到端吞吐量采用：

```text
THR_E2E(t) = min(THR_DL(t), THR_UL(t))
```

端到端误块率采用：

```text
BLER_E2E(t) = max(BLER_DL(t), BLER_UL(t))
```

通过该步骤，本发明不再将上下行链路视作割裂的两个仿真结果，而是形成可直接映射业务体验的统一评价值。

#### 步骤 S5：生成最坏工况干扰包络并进行搜索
构建生成模型 `G(z, c)`，其中 `z` 为随机潜变量，`c` 为可解释控制码；由生成模型输出归一化时变干扰包络 `jamAgg(t)`。之后在 `z`、`c` 以及干扰尺度因子空间中，通过搜索算法寻找使目标函数最劣的解。

优选地，目标函数可写为：

```text
f = -meanThr + W_outage * outage + W_bler * meanBler + W_energy * penalty
```

其中，`meanThr` 为平均吞吐量，`outage` 为中断率，`meanBler` 为平均误块率，`penalty` 为干扰能量和占空比约束项。

优选地：

1. 生成模型采用带可解释控制码的信息最大化生成模型。
2. 搜索算法采用遗传算法。
3. 搜索目标可选为下行链路、上行链路或端到端指标。

该步骤的有益效果在于：最坏工况不是由测试人员凭经验主观指定，而是由模型自动在可解释潜空间中搜索得到，更有利于发现隐藏失效路径。

#### 步骤 S6：在最坏工况下进行双链路协同评估
将步骤 S5 生成的最坏工况干扰包络回灌到双链路评估模型中，对上下行链路进行联合求解。优选地，上行链路沿用下行链路选择出的服务卫星和网关卫星，以便在同一路由约束下进行协同评价。

在该步骤中还考虑抗干扰动作延迟和空域零陷深度等参数。优选地，当检测到干扰功率超过阈值且持续时间达到预设延迟后，触发抗干扰保护，并按零陷深度对有效干扰功率进行衰减。

通过该步骤，可以量化“未保护”和“已保护”两种状态之间的差异，从而把抗干扰动作是否及时、是否有效也纳入评估范围。

#### 步骤 S7：进行功率对齐的时频图干扰识别
基于步骤 S6 输出的服务信号功率、共信道干扰功率、干扰功率和噪声功率，构造功率对齐的 IQ 快照；再对所述 IQ 快照执行短时傅里叶变换，生成统一尺寸的时频图。

将所述时频图输入轻量卷积神经网络，输出干扰类型标签。优选地，分类标签至少包括：

1. 无干扰。
2. 单音干扰。
3. 部分带噪声压制干扰。
4. 调制型干扰。

优选地，系统还根据事件标签自动挑选关键时间步，导出关键帧时频图和关键帧拼图。

该步骤的有益效果在于：平台不仅知道“性能下降了多少”，还能知道“是什么类型的干扰导致了性能下降”，从而提高对研发、测试和专利撰写的支撑价值。

#### 步骤 S8：输出实验室评估结果和附图素材
输出内容优选包括：

1. 基线工况与最坏工况的平均吞吐量和中断率。
2. 上行、下行与端到端评价结果。
3. 训练样例拼图。
4. 测试集混淆矩阵。
5. 仿真关键帧时频图。
6. 配置文件、结果文件和摘要文本。

通过该步骤，实验室评估不再只停留于屏幕展示，而是能形成标准化的专利附图、试验摘要和工程留档文件。

### 4.4 关键技术点
本发明的关键技术点至少包括：

1. 以低轨星座时变几何为基础的双链路 EMC 统一评估框架。
2. 将频率复用、共信道干扰、外部干扰和星间链路路由同时纳入同一评价闭环。
3. 以可解释生成模型加搜索算法自动生成最坏工况干扰包络。
4. 以功率对齐 IQ 快照和时频图分类结果作为链路退化解释层。
5. 以端到端吞吐量和端到端误块率作为双链路联合评价核心指标。

### 4.5 可选的替代实施方式
在不脱离本发明总体思想的前提下，可采用如下替代方式。

1. 搜索算法可由遗传算法替换为粒子群、贝叶斯优化或其他黑箱优化算法。
2. 生成模型可由信息最大化生成模型替换为变分自编码器、扩散模型或其他具备包络生成能力的模型。
3. 干扰识别网络可由轻量卷积神经网络替换为其他轻量级图像分类网络。
4. 干扰源可由干扰卫星扩展为机载、地面或同车其他射频发射源。
5. 结果输出除面向纯仿真外，还可外接天线因子、线损、前放增益和其他补偿项，实现与实验室端口测量值或场强值之间的换算。

## 五、具体实施例及其他有助于理解本技术的资料

### 5.1 实施例参数
本实施例采用如下参数进行说明。

1. 纪元时刻为 2026 年 1 月 23 日 00:00:00（UTC）。
2. 采样时间为 10 秒。
3. 星座高度为 1200 km，轨道倾角为 53 度。
4. 轨道面数量为 12，每轨卫星数量为 8，总卫星数量为 96。
5. 频率复用系数为 4，最小仰角门限为 5 度。
6. 下行中心频率为 1.5 GHz，上行中心频率为 1.6 GHz。
7. 干扰卫星数量为 6。
8. 抗干扰动作延迟为 30 秒，零陷深度为 40 dB。
9. 端到端最低吞吐量要求为 20 Mbps。

### 5.2 现有仿真平台导出的实施例结果
基于现有平台导出的摘要结果，本实施例得到如下指标。

1. 下行链路基线平均吞吐量为 78.97 Mbps，最坏工况平均吞吐量为 75.97 Mbps；下行链路中断率由 0 上升至 4.42%。
2. 上行链路基线平均吞吐量为 75.69 Mbps，最坏工况平均吞吐量为 52.62 Mbps；上行链路中断率由 0 上升至 20.73%。
3. 端到端基线平均吞吐量为 73.99 Mbps，最坏工况平均吞吐量为 50.57 Mbps；端到端中断率由 0 上升至 21.65%。

由此可以看出，本发明能够自动识别出在当前参数配置下，上行链路和端到端性能对最坏工况更敏感，从而避免仅以单一链路或单一端口做结论所带来的偏差。

## 六、本发明的关键点和欲保护点是什么？是否有替代方案同样能实现发明目的？
本发明的关键点和欲保护点如下。

1. 不是普通的车载卫星通信流程，而是将“时变轨道几何 + 双链路 EMC 评估 + 最坏工况生成 + 干扰识别”闭环打通。
2. 不是简单枚举干扰参数，而是利用生成模型在潜空间中产生时变干扰包络，再用搜索算法自动寻找最坏工况。
3. 不是只看单链路性能，而是通过 `min{DL, UL}` 和 `max{BLER_DL, BLER_UL}` 构造端到端评价值，直接定位业务瓶颈。
4. 不是只输出性能退化数值，而是输出与性能退化对应的干扰类型和关键帧时频图，具备更强的解释力和工程可用性。
5. 不是单纯软件算法描述，而是给出面向实验室评估平台的完整模块划分、输入输出关系和实施例参数。

替代方案同样可以实现发明目的，但仍应落入本发明保护范围，包括：

1. 使用不同的生成模型产生时变干扰包络。
2. 使用不同的搜索算法寻找最坏工况。
3. 使用不同的时频变换方式和轻量分类网络进行干扰识别。
4. 将干扰源从卫星扩展到地面、车载或无人平台。

对比项 1：车载卫星通信系统及车辆。

技术区别描述：现有方案主要围绕车载卫星通信终端结构、收发流程、设备集成和车辆应用展开，重点是“如何通信”或“如何接入卫星”，并未公开基于低轨时变场景的双链路 EMC 最坏工况生成、共信道干扰统一建模、端到端评价和时频图干扰识别这一整套实验室评估闭环。

相关程度：高。

对比项 2：基于卫星通信的车联网数据压缩方法、装置、设备和介质。

技术区别描述：现有方案解决的是业务传输中的数据压缩和通信成本问题，不涉及电磁兼容最坏工况生成，也不涉及上行、下行和端到端的统一 EMC 评估。

相关程度：中。

对比项 3：一种车载天线故障现场测试诊断装置。

技术区别描述：该方案解决的是现场信号通路衰减检测和故障诊断，属于现场测量装置技术路线；本发明解决的是研发和实验室阶段对车载低轨卫星通信系统进行时变场景 EMC 仿真、最坏工况搜索和干扰识别的技术问题，二者应用阶段、对象和技术实现均不同。

相关程度：中。

对比项 4：一种级联神经网络与场源等效联合的高速磁浮列车电磁环境建模方法。

技术区别描述：该方案偏向静态或准静态电磁场建模与神经网络近场预测，侧重等效源和屏蔽效能；本发明侧重低轨卫星时变场景下的双链路功率评估、生成式最坏工况搜索和时频图干扰分类，不涉及相同的场景组织方式和评价闭环。

相关程度：中。

综合判断：现有资料中虽然分别存在车载卫星通信、现场故障检测、近场电磁建模和神经网络分类等方案，但尚未发现将“低轨时变几何、双链路 EMC 评估、生成式最坏工况搜索、功率对齐时频图干扰识别和实验室结果导出”以本发明方式进行组合的公开方案。本发明适合从“方法 + 系统 + 设备/介质”三个层面进行专利布局。
